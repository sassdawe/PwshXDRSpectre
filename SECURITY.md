# Security Policy

## Supported Versions

Security fixes are provided for the latest `main` branch and the latest published release.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Report privately by contacting project maintainers with:

1. A clear description of the issue.
2. Impact assessment.
3. Reproduction steps or proof of concept.
4. Suggested mitigation, if available.

## Response Targets

1. Initial acknowledgment: within 3 business days.
2. Triage decision: within 7 business days.
3. Critical vulnerability fix target: within 48 hours after confirmation.

## Disclosure

Coordinated disclosure is preferred. Public disclosure should wait until a fix or mitigation is available.
